1.
A.default（缺省，不写）：类定义时前面未加任何修饰符，表示同一个包中可见。
B.public：修饰类时表示该类能被项目中所有类可见
C.abstract：表示是抽象类
D.final：表示类不可以被继承
E.scrictpf:(java关键字) 当对一个类或接口使用 strictfp 关键字时，该类中的所有代码，包括嵌套类型中的初始设定值和代码，都将严格地进行计算。严格约束意味着所有表达式的结果都必须是 IEEE 754 算法对操作数预期的结果，以单精度和双精度格式表示
2.
A.default (即缺省，什么也不写）: 在同一包内可见，不使用任何修饰符。使用对象：类、接口、变量、方法。
B.private : 在同一类内可见。使用对象：变量、方法。 注意：不能修饰类（外部类）
C.public : 对所有类可见。使用对象：类、接口、变量、方法
D.protected : 对同一包内的类和所有子类可见。使用对象：变量、方法。 注意：不能修饰类（外部类）。

3.
A.关键字final
I.可用于修饰:成员变量,非抽象类(不能与abstract同时出现),非抽象的成员方法,以及方法参数
II.final方法:不能被子类的方法重写,但可以被继承;
III.final类:表示该类不能被继承,没有子类;final类中的方法也无法被继承.
IV.final变量:表示常量,只能赋值一次,赋值后不能被修改.final变量必须定义初始化;
V.final不能用于修饰构造方法;
VI.final参数:只能使用该参数,不能修改该参数的值;
B.关键字:static
I.可以修饰成员变量和成员方法,但不能修饰类以及构造方法;
II.被static修饰的成员变量和成员方法独立于该类的任何对象。也就是说，它不依赖类特定的实例，被类的所有实例共享
III.static变量和static方法一般是通过类名直接访问,但也可以通过类的实例来访问(不推荐这种访问方式)
IV.static变量和static方法同样适应java访问修饰符.用public修饰的static变量和static方法,在任何地方都可以通过类名直接来访问,但用private修饰的static变量和static方法,只能在声明的本类方法及静态块中访问,但不能用this访问,因为this属于非静态变量.
C.关键字abstract
I.采用 abstract 方法的类本来就是抽象类，并且必须声明为 abstract。
abstract 类不能实例化。
II.仅当 abstract 类的子类实现其超类的所有 abstract 方法时，才能实例化 abstract 类的子类。这种类称为具体类，以区别于 abstract 类。
III.如果 abstract 类的子类没有实现其超类的所有 abstract 方法，该子类也是 abstract 类。
IV.abstract 关键字不能应用于 static、private 或 final 方法，因为这些方法不能被重写，因此，不能在子类中实现。
V.final 类的方法都不能是 abstract，因为 final 类不能有子类。
